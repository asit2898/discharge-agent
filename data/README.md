# Data

## `hero_patient/` — the demo fixture

One multi-specialist discharge encounter with **three planted discrepancies**, built
in the same schema shape as the provided `synthetic-ambient-fhir-25` dataset.

### Two-stage pipeline (the same shape Abridge used)

1. **Base — Synthea.** A real patient generated by
   [synthetichealth/synthea](https://github.com/synthetichealth/synthea):
   **Philomena Goodwin, 82F, Boston** (cardiac history — prasugrel, statin,
   beta-blocker; HTN; T2DM). We reuse her **real RxNorm-coded medications**, so no
   drug code is invented.
2. **Curated overlay.** A surgical admission across **three prescribers** with the
   transcript/note/AVS authored on top. Synthea can't produce spoken contradictions
   or an orphaned course, so those are layered on — exactly the gap the agent exists
   to catch.

### The scenario
82F admitted for **laparoscopic cholecystectomy**; post-op **intra-abdominal
infection** + **AKI**. Three prescribers:
- **Surgeon** (Dr. Nair) — did the operation; held prasugrel peri-op, resumes at d/c.
- **Hospitalist** (Dr. Whitfield) — manages home meds.
- **Infectious Disease** (Dr. Bell) — runs the antibiotic course.

### The three planted discrepancies (see `ground_truth.json`)
| ID | Type | Med | The bug |
|----|------|-----|---------|
| **D1** | `discontinued_but_listed` | ramipril 2.5 | Hospitalist stopped it for AKI (says so in transcript + note) but it's **still on the discharge order set**. |
| **D2** | `duplicate_therapy` | labetalol + metoprolol succ ER | Beta-blocker was **switched**, but the old labetalol was never removed — **both** on the discharge list. |
| **D3** | `orphaned_course` | amoxicillin-clavulanate | ID ordered a 7-day course; 3 days remain at discharge but it's **missing from the discharge order set**. |

Plus **6 agreement cases** that must **not** be flagged (atorvastatin, metformin,
prasugrel-resumed, aspirin, the metoprolol switch-target, hydrocodone/acetaminophen) —
these exist to measure precision, not just recall.

Each discrepancy is a conflict between the **discharge order set** (the buggy pending
list) and the **note/transcript** (the physician plan of record). Per our authority
rule, the note/order is where the agent defaults its suggested resolution; the
transcript only earns the flag.

### Files
- `hero_patient.json` — the record (patient_context, encounter_fhir, transcript, note, AVS)
- `ground_truth.json` — planted discrepancies + agreement cases (eval key)
- `source_provenance.json` — real Synthea patient identity + reused RxNorm codings
- `build.py` — reproducible builder for the two files above

### Reproduce
```bash
# base bundle (deterministic — seed 4242)
cd synthea-tool && ./run_synthea -s 4242 -p 40 Massachusetts   # -> output/fhir/Philomena945_*.json
# fixture
cd discharge-agent/data/hero_patient && python3 build.py
```

> The full 28 MB Synthea bundle is **not committed** (regenerable from the seed
> above). `source_provenance.json` captures the identity + codings we actually used.

### Deviation from the provided schema
The fixture uses `medicationCodeableConcept` (inline drug name) instead of the
dataset's `medicationReference` (uuid), so each order is self-contained and carries
its own resolvable name. Discharge orders are marked with a `Discharge` category
label — the dataset only has `Inpatient`/`Community`; the discharge order set is
what Epic pre-loads for reconciliation, so we model it explicitly.
